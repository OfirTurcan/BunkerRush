מה עבד טוב

  • useFrame ← לא re-renders — כל הפיזיקה, הקולישן והאנימציות רצות ב-frame loop. Zustand selectors מונעים re-renders מיותרים.
  • Zustand Store שטוח — כל ה-state וה-actions במקום אחד, ללא Context. קל לבדוק, קל לנפות.
  • Web Audio API — ללא קבצים — כל הצלילים פרוצדוראליים. אין תלות ב-CDN, אין בעיות CORS, טעינה מיידית.
  • AABB פשוט — קולישן מלבני ללא פיזיקה מורכבת. מהיר, קריא, קל לבדיקה ביחידות.
  • CanvasTexture פעם אחת — 4 טקסטורות כרזה נוצרות once, משוכפלות על עשרות בניינים. אפס עומס GPU נוסף.

שגיאות ולקחים

שגיאות שנתקלנו בהן

  • מוקשים נעלמים: טווח Z כיסה רק 390 מ' מתוך 800
  • פס ההתקדמות: Math.min(raw,1) גרם ל-progress=0 בשלבים 2-5
  • כיפת ברזל: obstacles ממשיכים לנוע כשהם בלתי-נראים
  • אנימציית כפיפה: חישוב חד-פעמי ב-render, לא ב-useFrame

לקחים לעתיד

  • להגדיר גבולות עולם מראש, לפני כתיבת spawn logic
  • לא לשמור state ב-module-level variables ב-components
  • להוסיף בדיקות אינטגרציה מוקדם יותר
  • טיפול מיוחד ב-try-catch שמסתיר errors מ-Audio API
  • כל אנימציה ← useFrame. לא render prop.
